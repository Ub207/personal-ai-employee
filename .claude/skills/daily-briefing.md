# Skill: Daily Briefing

## Purpose
Generate a structured daily CEO briefing and save it to `/Briefings/`.
This transforms raw activity data into an executive summary the owner can
act on immediately.

## Trigger
Run this skill:
- Every morning (or when the user asks for a briefing)
- After a Sunday night weekly audit

## Steps

1. **Read `Company_Handbook.md`** — load business goals, revenue targets, key metrics.

2. **Scan `/Done`** — list all files completed in the last 7 days (or since last briefing).

3. **Scan `/Pending_Approval`** — list any items still awaiting approval with their age.

4. **Scan `/Needs_Action`** — list any items still pending (backlog).

5. **Check financials** — if `Bank_Transactions.md` or similar exists, summarise revenue
   vs. monthly goal from the handbook.

6. **Identify bottlenecks** — tasks in `/Done` that took > 2 days from creation to completion.

7. **Check subscriptions** — scan `Company_Handbook.md` "Active Subscriptions" section;
   flag any unused > 30 days (use last activity dates if tracked).

8. **Write the briefing** to `/Briefings/YYYY-MM-DD_Briefing.md` using the template below.

9. **Update `Dashboard.md`** — add a row to the Recent Activity Log noting the briefing was generated.

## Output Template

```markdown
---
generated: {ISO_TIMESTAMP}
period: {START_DATE} to {END_DATE}
type: daily_briefing
---

# {WEEKDAY} Morning CEO Briefing — {DATE}

## Executive Summary
{1-2 sentence plain-English summary of the week}

## Completed This Period
{bullet list of /Done items}

## Pending Approvals ({COUNT})
{table of pending items with age and required action}

## Backlog ({COUNT} items in /Needs_Action)
{list any items > 24 hours old}

## Financial Snapshot
| Metric | Value | Target | Status |
|--------|-------|--------|--------|
| MTD Revenue | ${MTD} | ${GOAL} | {on/off track} |
| Open Invoices | {N} | 0 | — |

## Bottlenecks
{tasks that took longer than expected}

## Proactive Suggestions
{cost savings, unused subscriptions, deadlines approaching}

---
*Generated by AI Employee v0.1*
```

## Rules
- Never fabricate financial data — only report what exists in vault files.
- If no financial data exists, write "No financial data available — update Bank_Transactions.md."
- Keep the briefing under 1 page (100 lines) for readability.

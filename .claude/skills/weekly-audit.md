# Skill: Weekly Audit

## Purpose
Generate the Monday Morning CEO Briefing — a comprehensive weekly snapshot of
all business activity, financials, social media, system health, and subscription
status. Saves to `/Briefings/YYYY-MM-DD_Weekly_CEO_Briefing.md`.

## Trigger
Run this skill:
- Every Monday morning (or when the user asks for a weekly review)
- After a significant week of activity
- When the `daily_briefing_scheduler.py` triggers a weekly run

## Steps

1. **Read `Company_Handbook.md`** — load revenue targets, active subscriptions,
   and business goals.

2. **Scan `/Done`** for all items completed in the last 7 days:
   - Count items by type (email, invoice, task, social post, etc.)
   - Note any item that was in `/Needs_Action` for more than 24 hours

3. **Scan `/Pending_Approval`** — list all items awaiting sign-off:
   - Include age (hours/days) for each item
   - Flag any items > 48 hours old as overdue

4. **Read Odoo financial data** from `Logs/odoo_actions.json`:
   - Count invoices created this week
   - Note any revenue queries or accounting summaries
   - If Odoo MCP server is available, call `get_revenue_summary` for the week
   - If Odoo MCP server is available, call `get_accounting_summary` for overdue invoices

5. **Read social media activity** from `Logs/social_actions.json`:
   - Count posts per platform (Twitter, Facebook, Instagram)
   - Flag any posting errors

6. **Read audit events** from `Logs/audit_*.json`:
   - Total actions vs errors
   - Break down by component/source
   - Highlight any error spikes

7. **Check system health** from `vault/health_status.json`:
   - List watcher status (running/down/disabled)
   - Flag any disabled processes

8. **Subscription audit** — parse "Active Subscriptions" table in `Company_Handbook.md`:
   - Flag any service with last-used date > 30 days ago

9. **Generate the briefing** using the template below.
   Save to `/Briefings/YYYY-MM-DD_Weekly_CEO_Briefing.md`.

10. **Update `Dashboard.md`** — add a row to Recent Activity noting the briefing.

11. **Report back** a summary: key metrics and the file path.

## Rules
- Never fabricate financial numbers — only report what exists in log files.
- If no Odoo data exists, say so clearly and suggest connecting the MCP server.
- Keep the briefing factual and actionable — no filler text.
- Always include the "Recommended Actions" section at the end.
- If health_status.json shows a DISABLED watcher, escalate it as a critical action.
